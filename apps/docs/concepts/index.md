# Key Concepts in AuthHero

This section explains the core concepts and terminology used in AuthHero.

## Core Concepts

- **[Tenants](/concepts/tenants)** - Logical isolation of users, applications, and authentication settings
- **[Applications](/concepts/applications)** - Client applications that use AuthHero for authentication
- **[Connections](/concepts/connections)** - Authentication methods available to users
- **[Users](/concepts/users)** - Individuals who authenticate through AuthHero
- **[Organizations](/concepts/organizations)** - Groups of users with specific configurations and access controls
- **[Resource Servers](/concepts/resource-servers)** - APIs protected by AuthHero
- **[Tokens](/concepts/tokens)** - Various types of authentication and authorization tokens
- **[Hooks](/concepts/hooks)** - Extensibility mechanism to customize authentication flows
- **[Adapters](/concepts/adapters)** - Data storage and platform integrations

For a deeper understanding of how these concepts work together, see the [Architecture](/architecture) and [Security Model](/security-model) documentation.
